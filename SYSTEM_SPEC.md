# Daily Todo App - システム仕様書

## 📋 概要

Daily Todo App は、毎日のタスク管理とメモ機能を備えたシンプルなWebアプリケーションです。
ユーザーは日々のタスクをチェック形式で管理し、メモを記録することができます。
すべてのデータはブラウザの LocalStorage に保存され、サーバー側の処理は不要です。

---

## 🎯 主な機能

### 1. **タスク管理機能**
- 5 つのカスタマイズ可能なタスク項目を表示
- 各タスクには ON/OFF のトグルスイッチ
- チェック状態は LocalStorage に自動保存
- 全タスク完了時に祝いメッセージを表示
- 完了状態で背景色が変化

### 2. **メモ機能**
- テキストエリアでメモ記入可能
- メモは LocalStorage に永続保存
- ブラウザ再起動後も保持される

### 3. **履歴機能**
- 過去 30 日間のタスク完了状態を記録
- 下からスライドアップするパネルで表示
- 日付ごとにタスク完了状況を確認可能

### 4. **デイリーリセット機能**
- 日付が変わるとタスク状態がリセット
- 前日のタスク完了状態は自動的に履歴に保存
- 新しい日は全タスク未完了状態で開始

## 💾 LocalStorage データ構造

### 1. **checked_tasks**
```json
{
  "タスク名1": true,
  "タスク名2": false,
  "タスク名3": true
}
```
- **型：** Object
- **用途：** 本日のタスクチェック状態
- **有効期間：** 日付変更まで

### 2. **persistent_memo**
```
テキスト形式のメモ内容
```
- **型：** String
- **用途：** ユーザーのメモ記録
- **有効期間：** 削除されるまで永続

### 3. **task_history**
```json
{
  "2024-01-15": { "タスク名1": true, "タスク名2": false },
  "2024-01-14": { "タスク名1": true, "タスク名2": true },
  ...
}
```
- **型：** Object (キー：日付 YYYY-MM-DD 形式)
- **用途：** 過去 30 日のタスク完了履歴
- **最大保持数：** 30 日分

### 4. **last_visit_date**
```
2024-01-15
```
- **型：** String (YYYY-MM-DD 形式)
- **用途：** 最後にアプリを訪問した日付追跡
- **用途：** 日付変更の検知に使用

## 🎨 UI/UX 仕様

### 1. **色彩設計**
| 項目 | 色コード | 用途 |
|------|---------|------|
| デフォルト背景 | #f6f8fa | 通常時の背景 |
| 完了時背景 | #87CEEB | 全タスク完了時 |
| トグル ON | #007bff | スイッチ ON 時 |
| トグル OFF | #afb8c1 | スイッチ OFF 時 |

### 2. **レイアウト**
- **デバイス対応：** モバイルファースト設計
- **ビューポート制限：** `maximum-scale=1.0, user-scalable=no`
- **最大幅：** 600px（タブレット以上での表示最大幅）
- **フォント：** システムフォント（-apple-system, BlinkMacSystemFont, Segoe UI, Roboto）

### 3. **主要コンポーネント**

#### ヘッダーエリア
- アプリタイトル：「Daily Tasks」
- 本日の日付表示（YYYY/MM/DD 形式）

#### タスク一覧
- 各タスクにトグルスイッチを配置
- グラデーション風の切り替えアニメーション（0.3s）
- タスク名は左寄せ、スイッチは右寄せ

#### メモ欄
- 親指操作しやすい位置に配置
- 高さ：90px（スマートフォンで適切な大きさ）
- 入力時にリアルタイム保存

#### 履歴ボタン
- 下部に配置、親指で押しやすい設計
- クリックで下からスライドアップするパネル表示

#### 履歴パネル
- ビューポート高さの 80% を占有
- ボトムシート UI パターン
- スムーズなスライドアニメーション（0.4s cubic-bezier）

## 🔄 ユーザーフロー

### 1. **初回訪問時**
```
1. LocalStorage 初期化
2. 本日の日付を記録 (last_visit_date)
3. 全タスクを未完了状態で表示
4. 保存されたメモがあれば復元
```

### 2. **タスクチェック時**
```
1. トグルスイッチクリック
2. checked_tasks 更新
3. LocalStorage に保存
4. 全タスク完了判定
5. 完了なら背景色変化 + メッセージ表示
```

### 3. **メモ入力時**
```
1. テキスト入力
2. input イベント発火
3. persistent_memo に保存
```

### 4. **日付変更検知**
```
1. ページロード
2. last_visit_date と 本日の日付を比較
3. 異なれば前日のタスク状態を履歴に保存
4. checked_tasks をクリア（リセット）
5. 履歴は最新 30 日分のみ保持
```

### 5. **履歴表示**
```
1. 「履歴を確認する」ボタンクリック
2. task_history を日付昇順で取得
3. パネルをスライドアップして表示
4. 左上の日付、右側に完了数表示
```

## ✅ テスト仕様

### テスト実行方法
```bash
npm test                # ヘッドレスモードで実行
npm run test:headed     # ブラウザ表示で実行
```

### テストケース一覧

| # | テスト名 | 説明 |
|----|---------|------|
| 1 | ページが正しくロードされる | タイトルと日付表示を確認 |
| 2 | タスク一覧が表示される | 5 個のタスク項目が描画される |
| 3 | タスクのチェックボックスが機能する | 最初のタスクをチェック/解除 |
| 4 | 複数のタスクをチェックできる | 最初の 3 つのタスクをチェック |
| 5 | 全てのタスクを完了するとお祝いメッセージが表示される | すべてチェック時にメッセージ表示 |
| 6 | メモが保存される | メモ入力後、再ロード時に保持 |
| 7 | メモは異なるセッションでも保存される | LocalStorage 確認 |
| 8 | 履歴パネルが開閉する | パネルの class 切り替え確認 |
| 9 | チェック状態がLocalStorageに保存される | LocalStorage の checked_tasks 確認 |
| 10 | ページリロード後もチェック状態が保持される | リロード後の状態保持確認 |
| 11 | 背景色がタスク完了状態で変わる | `is-complete` class による背景色変化 |
